OVERVIEW
================================================================================
The Strategy Research tab is now operational as a hybrid workflow:

1. Cloud mode: Uses authenticated backend APIs for strategy/version/run lifecycle.
2. Local mode: Works on desktop even when backend auth is unavailable.
3. Bridge readiness mode: Paper/live preconditions can be checked through
  governance readiness and connector APIs before activation.

Local mode includes:
- Local strategy persistence in desktop SQLite workspace tables.
- Local historical data download and cache on user PC.
- Local backtest execution through Electron main process service.
- Run history, metrics, equity curve rendering, and trade preview in UI.

This resolves prior usability blockers where missing auth could prevent create/save/run.

STORE BEHAVIOR (HYBRID FALLBACK)
================================================================================
File: apps/desktop/src/renderer/store/strategyResearchStore.ts

Key behavior:
- Attempts cloud APIs first for strategy/read/save/runs.
- Detects auth failures and falls back to local workspace mode.
- Persists local strategies/versions/runs/comparison notes to desktop SQLite storage.
- Normalizes backend and local run/version payloads for shared UI rendering.
- Supports local historical download and local run execution via preload bridge.

Sync modes:
- syncMode=cloud when backend path succeeds.
- syncMode=local when fallback path is active.

================================================================================
USER WORKFLOWS NOW ENABLED
================================================================================
Workflow 1: Local-first strategy creation
1. Open STRATEGY RESEARCH tab.
2. Click + NEW in sidebar.
3. Enter name/description and create strategy.
4. Edit script, universe, and assumptions.
5. Save version.

Workflow 2: Download history to PC
1. In editor, set universe symbols (for example: AAPL, MSFT, SPY).
2. Click Download Historical Data.
3. Data downloads/caches locally and reports downloaded vs cache hits.

Workflow 3: Run desktop-local backtest
1. Select execution mode desktop-local.
2. Click Run Backtest.
3. Review run in Run History.
4. Open Details for metrics, curve, and trades.

Workflow 4: Cloud-backed run (when authenticated)
1. Sign in and keep backend session valid.
2. Select execution mode backend.
3. Enqueue run through backend API and inspect returned run state.
